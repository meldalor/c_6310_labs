## Задание

Вам необходимо разработать библиотеку `txtfile` в соответствии с одним из четырёх вариантов (вариант выдаётся преподавателем или выбирается по списку).

## Библиотека `txtfile`

Библиотека должна содержать следующие обязательные функции:

```c
char    **read_file(const char *filename);
void    free_lines(char **lines);
void    print_lines(char **lines);
```

### Назначение обязательных функций

- `read_file(const char *filename)` – считывает существующий текстовый файл и возвращает его содержимое в виде массива строк, заканчивающегося `NULL`. При ошибке возвращает `NULL`;
- `free_lines(char **lines)` – освобождает память, выделенную под массив строк;
- `print_lines(char **lines)` – выводит массив строк в `stdout`.

В зависимости от варианта добавляются дополнительные функции (см. варианты заданий).

---

## Программа `main.c`

Файл `main.c` содержит демонстрацию работы библиотеки:

- получает путь к существующему текстовому файлу через параметры командной строки (`argv`);
- получает параметры операции через параметры командной строки (`argv`);
- проверяет количество аргументов (`argc`) и при ошибке выводит сообщение `Usage`;
- считывает файл через `read_file`;
- вызывает функции библиотеки `txtfile` по своему варианту;
- выводит исходное содержимое файла и результаты через функции `libft` (`ft_putchar_fd`, `ft_putstr_fd`, `ft_putnbr_fd`, `ft_putendl_fd`);
- освобождает всю выделенную память через `free_lines`.

Примеры запуска:

```bash
./grep_file input.txt apple
./cut_file input.txt 2 4
./replace_file input.txt cat fox
./analyze_file input.txt
```

Запрещается использование стандартных функций из `<stdio.h>`, `<stdlib.h>`, `<string.h>` и любых других, не входящих в `libft` (кроме разрешённых системных вызовов `open`, `close`, `read`, `write`, `malloc`, `free`, которые могут использоваться только внутри `libft` и `txtfile`).

## Общие требования ко всем вариантам

- Код должен компилироваться с флагами `-Wall -Wextra -Werror`.
- Не должно быть утечек памяти (проверка через `valgrind`).
- Все указатели, полученные через `malloc`/`ft_calloc`, должны освобождаться.
- Функции библиотеки должны корректно обрабатывать ошибки:
  - `NULL` в аргументах;
  - отсутствие файла;
  - невозможность открыть файл;
  - невозможность выделить память;
  - пустой файл.
- В случае ошибки функции должны возвращать `NULL` или код ошибки.
- Работа ведётся **только с существующим текстовым файлом**, путь к которому передаётся через параметры командной строки (`argv`).
- Библиотека должна работать с текстом построчно.

## Вариант 1. Поиск строк по подстроке (`grep`)

### Теория

Поиск по подстроке — это операция отбора строк текстового файла, содержащих заданную последовательность символов.

Если файл состоит из строк:

```text
apple pie
banana split
pineapple juice
orange tea
```

и выполняется поиск по подстроке `"apple"`, то результатом будут строки:

```text
apple pie
pineapple juice
```

### Описание

Реализуйте функцию поиска строк, содержащих указанную подстроку.  
В `main.c` необходимо считать файл, получить заданное слово из `argv`, выполнить поиск и вывести найденные строки.

### Функция для реализации

```c
char    **grep_lines(char **lines, const char *pattern);
```

### Назначение функции

- возвращает новый массив строк, содержащий только те строки, в которых найдена подстрока `pattern`;
- если совпадений нет, функция может вернуть пустой массив строк (`char **` с первым элементом `NULL`) или `NULL` — в зависимости от принятой в проекте логики, но это должно быть единообразно;
- исходный массив `lines` изменяться не должен.

### Пример использования в `main.c`

```c
if (argc != 3)
{
    ft_putendl_fd("Usage: ./grep_file <filename> <pattern>", 2);
    return (1);
}

char **lines = read_file(argv[1]);
char **result = grep_lines(lines, argv[2]);

print_lines(result);
// Результат:
// apple pie
// pineapple juice

free_lines(lines);
free_lines(result);
```

## Вариант 2. Вырезание строк по диапазону (`cut` по строкам)

### Теория

Операция вырезания позволяет извлечь часть содержимого файла.  
В данном варианте требуется вырезать строки по их номерам.

Если файл содержит строки:

```text
line 1
line 2
line 3
line 4
line 5
```

то при вырезании строк с 2 по 4 результат будет:

```text
line 2
line 3
line 4
```

### Описание

Реализуйте функцию выделения диапазона строк из файла.  
В `main.c` необходимо считать файл, получить индексы из `argv`, вырезать строки по указанным индексам и вывести результат.

Нумерацию строк рекомендуется считать **с 1**.

### Функция для реализации

```c
char    **cut_lines(char **lines, int start, int end);
```

### Назначение функции

- возвращает новый массив строк, содержащий строки с номера `start` по номер `end` включительно;
- если диапазон некорректен (`start <= 0`, `end < start`, выход за границы файла), функция должна корректно обработать ошибку;
- исходный массив `lines` изменяться не должен.

### Пример использования в `main.c`

```c
if (argc != 4)
{
    ft_putendl_fd("Usage: ./cut_file <filename> <start> <end>", 2);
    return (1);
}

char **lines = read_file(argv[1]);
char **result = cut_lines(lines, ft_atoi(argv[2]), ft_atoi(argv[3]));

print_lines(result);
// Результат:
// line 2
// line 3
// line 4

free_lines(lines);
free_lines(result);
```

## Вариант 3. Замена подстроки во всём файле

### Теория

Замена подстроки — это операция, при которой во всех строках текста все вхождения одной подстроки заменяются другой.

Например, если исходный текст:

```text
cat is here
black cat
dog and cat
```

после замены `"cat"` на `"fox"` результат будет:

```text
fox is here
black fox
dog and fox
```

### Описание

Реализуйте функцию, которая создаёт новый массив строк, где во всём тексте каждая подстрока `old` заменяется на `new_str`.

В `main.c` необходимо считать файл, получить заменяемую и новую подстроки из `argv`, выполнить замену и вывести результат.

### Функция для реализации

```c
char    **replace_substr(char **lines, const char *old, const char *new_str);
```

### Назначение функции

- возвращает новый массив строк, где все вхождения `old` заменены на `new_str`;
- если подстрока `old` не найдена, строки копируются без изменений;
- исходный массив `lines` изменяться не должен.

### Пример использования в `main.c`

```c
if (argc != 4)
{
    ft_putendl_fd("Usage: ./replace_file <filename> <old> <new>", 2);
    return (1);
}

char **lines = read_file(argv[1]);
char **result = replace_substr(lines, argv[2], argv[3]);

print_lines(result);
// Результат:
// fox is here
// black fox
// dog and fox

free_lines(lines);
free_lines(result);
```

## Вариант 4. Анализ текстового файла

### Теория

Текстовый файл можно анализировать по различным характеристикам:

- **количество строк** — число строк в файле;
- **количество символов** — общее число символов во всех строках;
- **количество слов** — число последовательностей символов, разделённых пробелами, табуляцией или переводами строки;
- **самая длинная строка** — строка с максимальной длиной.

Например, для текста:

```text
Hello world
This is text
abc
```

- строк: `3`
- символов (без учёта завершающего `\0`): `24`
- слов: `6`
- длина самой длинной строки: `12`

### Описание

Реализуйте функции анализа текстового файла.  
В `main.c` необходимо считать файл, переданный через `argv`, вычислить характеристики текста и вывести результаты.

### Функции для реализации

```c
int count_lines(char **lines);
int count_words(char **lines);
int count_chars(char **lines);
int longest_line_length(char **lines);
```

### Назначение функций

- `count_lines` — возвращает количество строк;
- `count_words` — возвращает количество слов;
- `count_chars` — возвращает количество символов во всех строках;
- `longest_line_length` — возвращает длину самой длинной строки.

Каждая функция возвращает вычисленное значение, а при ошибке — `-1`.

### Пример использования в `main.c`

```c
if (argc != 2)
{
    ft_putendl_fd("Usage: ./analyze_file <filename>", 2);
    return (1);
}

char **lines = read_file(argv[1]);

ft_putnbr_fd(count_lines(lines), 1);
ft_putchar_fd('\n', 1);

ft_putnbr_fd(count_words(lines), 1);
ft_putchar_fd('\n', 1);

ft_putnbr_fd(count_chars(lines), 1);
ft_putchar_fd('\n', 1);

ft_putnbr_fd(longest_line_length(lines), 1);
ft_putchar_fd('\n', 1);

free_lines(lines);
```

# Makefile

В `Makefile` должны быть цели:

- `all` — сборка программы;
- `clean` — удаление объектных файлов;
- `fclean` — удаление объектных файлов и исполняемого файла;
- `re` — пересборка.

### Пример `Makefile`

```make
NAME = txtfile
CC = gcc
CFLAGS = -Wall -Wextra -Werror -I./inc
LIBFT_DIR = ./libft
LIBFT = $(LIBFT_DIR)/libft.a

SRC_DIR = src
SRCS = $(SRC_DIR)/main.c \
       $(SRC_DIR)/read_file.c \
       $(SRC_DIR)/free_lines.c \
       $(SRC_DIR)/print_lines.c \
       $(SRC_DIR)/grep_lines.c
OBJS = $(SRCS:.c=.o)

all: $(NAME)

$(NAME): $(OBJS) $(LIBFT)
	$(CC) $(CFLAGS) $^ -o $@

$(LIBFT):
	make -C $(LIBFT_DIR)

%.o: %.c
	$(CC) $(CFLAGS) -c $< -o $@

clean:
	rm -f $(OBJS)
	make -C $(LIBFT_DIR) clean

fclean: clean
	rm -f $(NAME)
	make -C $(LIBFT_DIR) fclean

re: fclean all

.PHONY: all clean fclean re
```

**Примечание:** в примере `Makefile` указаны файлы для **варианта 1**. Для других вариантов замените `grep_lines.c` на соответствующие файлы вашего варианта.
