# Чек-лист для самопроверки

## Тестирование

### Ручное тестирование

Запустите программу несколько раз и проверьте корректность поведения потоков.

Пример компиляции и запуска:

```bash
gcc philosophers.c -Wall -Wextra -Werror -pthread -o philosophers
./philosophers
```

Убедитесь, что:

- создаются 5 потоков философов;
- философы переходят между состояниями `THINKING`, `HUNGRY`, `EATING`;
- выводятся сообщения о захвате и освобождении вилок;
- флаговый `enum` используется для отслеживания вилок в руках философа;
- программа завершается корректно после заданного числа итераций;
- при нескольких запусках программа не зависает.

### Проверка deadlock

Запустите программу несколько раз подряд:

```bash
./philosophers
./philosophers
./philosophers
```

Убедитесь, что программа не застревает в состоянии ожидания навсегда и во всех запусках корректно завершается.

### Проверка race condition

Проверьте корректность поведения потоков и отсутствие состояния гонки (race condition):

- вывод не содержит "ломаных" или частично перемешанных строк;
- один и тот же ресурс (вилка) не используется одновременно двумя философами;
- состояния философов изменяются последовательно и логично.

### Проверка утечек памяти

Используйте `valgrind`:

```bash
valgrind --leak-check=full ./philosophers
```

Убедитесь, что:

- нет утечек памяти;
- все выделенные блоки освобождены;
- все мьютексы уничтожены корректно.

## Чек-лист

- [ ] Программа компилируется без ошибок и предупреждений командой `gcc philosophers.c -Wall -Wextra -Werror -pthread -o philosophers`.
- [ ] В `Makefile` реализованы цели `all`, `clean`, `fclean`, `re` (если он присутствует).
- [ ] Создаются 5 потоков философов через `pthread_create()`.
- [ ] Для вилок используется 5 мьютексов.
- [ ] Состояние философа реализовано через обычный `enum`.
- [ ] Для отслеживания вилок в руках философа используется флаговый `enum`.
- [ ] Каждый философ корректно проходит состояния `THINKING`, `HUNGRY`, `EATING`.
- [ ] При захвате и освобождении вилок корректно обновляются флаги `PHILO_HAS_LEFT` и `PHILO_HAS_RIGHT`.
- [ ] Доступ к вилкам синхронизирован через `pthread_mutex_lock()` / `pthread_mutex_unlock()`.
- [ ] Реализовано решение, предотвращающее deadlock.
- [ ] Отсутствуют race condition при работе с общими ресурсами (вилки, состояние философов, вывод).
- [ ] Все потоки корректно завершаются, `main` ожидает их через `pthread_join()`.
- [ ] Все мьютексы корректно уничтожаются после завершения программы.
- [ ] Программа не использует глобальные переменные.
- [ ] Нет утечек памяти (`valgrind` показывает "All heap blocks were freed").
- [ ] Код отформатирован, нет лишних включений заголовочных файлов.

## Критерии оценивания (максимум 5 баллов)

| Критерий | Баллы |
|----------|-------|
| Созданы потоки философов через `pthread_create()`, программа корректно работает в многопоточном режиме | 1 |
| Для вилок используются мьютексы, доступ к общим ресурсам синхронизирован корректно | 1 |
| Использованы два перечисления: обычный `enum` для состояний философа и флаговый `enum` для вилок в руках философа | 1 |
| Реализовано решение без deadlock и race condition, все потоки завершаются корректно, `main` выполняет `pthread_join()` | 1 |
| Отсутствуют утечки памяти и корректно освобождаются/уничтожаются все ресурсы | 1 |
| **Итого** | **5** |

### Штрафы:

- Отсутствие обычного `enum` для состояний философа — **−1 балл**.
- Отсутствие флагового `enum` для вилок в руках философа — **−1 балл**.
- Отсутствие мьютексов или некорректная синхронизация — работа не принимается (**0 баллов**).
- Наличие deadlock или race condition при обычном запуске программы — **−2 балла**.
- Утечки памяти или неуничтоженные мьютексы — **−1 балл**.
- Использование глобальных переменных — **−1 балл**.
- Программа не компилируется с флагами `-Wall -Wextra -Werror` и `-pthread` — **−5 баллов**.
