## Теоретическая справка

### Потоки в C

Поток (`thread`) — это отдельная последовательность выполнения внутри одного процесса. Потоки одного процесса разделяют общее адресное пространство, глобальные переменные и другие ресурсы, но имеют собственный стек и собственный контекст выполнения.

В языке C для работы с потоками в Linux и POSIX‑совместимых системах обычно используется библиотека `pthreads` (`POSIX threads`). Для подключения необходимо использовать заголовочный файл:

```c
#include <pthread.h>
```

Программа в этой лабораторной работе должна компилироваться командой:

```bash
gcc philosophers.c -Wall -Wextra -Werror -pthread -o philosophers
```

Основные функции, которые понадобятся в этой лабораторной работе:

- `pthread_create()` — создание нового потока;
- `pthread_join()` — ожидание завершения потока;
- `pthread_mutex_init()` — инициализация мьютекса;
- `pthread_mutex_lock()` — захват мьютекса;
- `pthread_mutex_unlock()` — освобождение мьютекса;
- `pthread_mutex_destroy()` — уничтожение мьютекса после завершения работы.

### Мьютекс

Мьютекс (`mutex`) — это механизм взаимного исключения. Он нужен для защиты общего ресурса от одновременного доступа из нескольких потоков. Если один поток захватил мьютекс, остальные потоки должны ждать, пока он не будет освобождён.

В задаче «обедающие философы» общими ресурсами являются вилки. Каждая вилка должна быть защищена своим мьютексом, чтобы два соседних философа не могли использовать её одновременно.

### `enum` и флаговый `enum`

В этой лабораторной работе требуется использовать два перечисления.

Первое — обычный `enum` для логического состояния философа:

```c
typedef enum e_state
{
    THINKING,
    HUNGRY,
    EATING
}   t_state;
```

Такой `enum` задаёт одно состояние из нескольких возможных: философ либо думает, либо голоден, либо ест.

Второе — флаговый `enum` для отслеживания того, какие вилки философ сейчас держит в руках:

```c
typedef enum e_philo_flags
{
    PHILO_NONE      = 0,
    PHILO_HAS_LEFT  = 1 << 0,
    PHILO_HAS_RIGHT = 1 << 1
}   t_philo_flags;
```

Здесь значения можно комбинировать побитовыми операциями: философ может не держать ни одной вилки, держать только левую, только правую или сразу обе.

## Состояние гонки (race condition)

Состояние гонки (race condition) возникает, когда несколько потоков одновременно обращаются к общему ресурсу, и результат выполнения программы зависит от порядка их выполнения.

Например, если два потока одновременно изменяют одну и ту же переменную без синхронизации, итоговое значение может быть непредсказуемым.

В задаче «обедающие философы» состояние гонки может возникнуть, если:

- два философа одновременно пытаются взять одну и ту же вилку;
- состояние философа изменяется без защиты;
- вывод в консоль выполняется одновременно из нескольких потоков.

Для предотвращения race condition необходимо использовать механизмы синхронизации, такие как мьютексы (`pthread_mutex_t`), и гарантировать, что доступ к общим ресурсам происходит только в защищённых участках кода.

## Задача «обедающие философы»

Задача «обедающие философы» — это классическая задача синхронизации, предложенная Эдсгером Дейкстрой для демонстрации проблем совместного доступа к ресурсам и взаимных блокировок.

За круглым столом сидят 5 философов. Между каждыми двумя соседними философами лежит одна вилка, всего вилок тоже 5. Перед каждым философом стоит тарелка с блюдом, которое можно есть **только двумя вилками одновременно**. Поэтому для еды философ должен взять вилку слева от себя и вилку справа от себя.

Каждый философ циклически повторяет одно и то же поведение:

- думает;
- испытывает голод;
- пытается взять две вилки;
- ест;
- кладёт вилки обратно и снова начинает думать.

Проблема состоит в том, что вилки — общий ресурс. Если все философы одновременно возьмут по одной вилке и будут ждать вторую, возникнет взаимная блокировка (`deadlock`) — программа зависнет. Поэтому задача состоит не просто в создании потоков, а в корректной синхронизации доступа к вилкам так, чтобы философы могли есть и программа продолжала работать без deadlock.

## Задание

Необходимо разработать программу `philosophers`, моделирующую решение задачи «обедающие философы» с использованием потоков и мьютексов.

Программа должна быть реализована в `philosophers.c`.

Каждый философ должен быть реализован как отдельный поток. Каждая вилка должна быть реализована как отдельный мьютекс.

### Обязательные требования

Программа должна:

- создавать 5 потоков философов;
- использовать 5 мьютексов для вилок;
- моделировать циклическое поведение философа: `THINKING -> HUNGRY -> EATING -> THINKING`;
- использовать обычный `enum` для описания состояния философа;
- использовать флаговый `enum` для отслеживания того, какие вилки сейчас находятся в руках у философа;
- корректно синхронизировать доступ к вилкам через мьютексы;
- предотвращать взаимную блокировку (`deadlock`);
- выводить в консоль действия философов: философ думает, проголодался, взял левую вилку, взял правую вилку, ест, положил вилки.

### Что требуется реализовать

Минимально в проекте должны присутствовать:

```c
typedef enum e_state
{
    THINKING,
    HUNGRY,
    EATING
}   t_state;

typedef enum e_philo_flags
{
    PHILO_NONE      = 0,
    PHILO_HAS_LEFT  = 1 << 0,
    PHILO_HAS_RIGHT = 1 << 1
}   t_philo_flags;

typedef struct s_philosopher
{
    int             id;
    t_state         state;
    t_philo_flags   flags;
    pthread_t       thread;
    pthread_mutex_t *left_fork;
    pthread_mutex_t *right_fork;
}   t_philosopher;
```

Примерный набор функций:

```c
void    *philosopher_routine(void *arg);
void    think(t_philosopher *philo);
void    eat(t_philosopher *philo);
void    take_forks(t_philosopher *philo);
void    put_forks(t_philosopher *philo);
```

Допускается изменить имена структур и функций, если логика остаётся понятной и соответствует заданию.

## Требования к реализации

- Программа должна компилироваться без ошибок и предупреждений командой:

```bash
gcc philosophers.c -Wall -Wextra -Werror -pthread -o philosophers
```

- Реализация может находиться как в одном файле philosophers.c, так и быть разбита на несколько файлов (например: philosopher.c, utils.c, philosopher.h), если это улучшает читаемость и структуру кода.

При использовании нескольких файлов необходимо обеспечить корректную компиляцию проекта (например, через одну команду gcc или Makefile).

- Нельзя использовать глобальные переменные.
- Состояние каждого философа должно храниться в поле типа обычного `enum`.
- Информация о вилках, находящихся в руках философа, должна храниться в поле типа флагового `enum`.
- Доступ к вилкам должен выполняться **только** через мьютексы.
- Программа не должна завершаться аварийно при обычном запуске.
- Все созданные мьютексы должны быть корректно уничтожены через `pthread_mutex_destroy()` после завершения программы.
- Все созданные потоки должны быть корректно завершены, а `main` должен дождаться их через `pthread_join()`.
- Следует предусмотреть конечное число итераций для каждого философа, чтобы программу можно было корректно завершить и проверить.

### Как должен использоваться флаговый `enum`

Флаговый `enum` должен применяться именно по назначению — для хранения комбинации признаков, связанных с вилками:

- `PHILO_NONE` — философ не держит ни одной вилки;
- `PHILO_HAS_LEFT` — философ держит левую вилку;
- `PHILO_HAS_RIGHT` — философ держит правую вилку;
- `PHILO_HAS_LEFT | PHILO_HAS_RIGHT` — философ держит обе вилки.

При захвате вилки необходимо установить соответствующий флаг, а при освобождении — снять его.

### Требование по предотвращению deadlock

Необходимо реализовать **одно** из корректных решений предотвращения deadlock:

- философы с чётными номерами сначала берут левую вилку, затем правую, а с нечётными — наоборот;
- все философы захватывают вилки в едином порядке по номеру вилки;
- любое другое корректное решение, исключающее взаимную блокировку.

## Функцция `main`

Функция `main()` должна содержать демонстрацию работы программы:

- создание массива мьютексов для вилок;

- инициализацию всех структур;
- запуск потоков через `pthread_create()`;
- ожидание завершения потоков через `pthread_join()`;

Можно использовать фиксированное количество философов (`5`) и фиксированное количество циклов еды/размышления, чтобы не усложнять лабораторную работу разбором аргументов командной строки.
