---

  Теоретическая справка

   Указатели на функции
Указатель на функцию хранит адрес кода функции и позволяет вызывать функцию косвенно. Это даёт возможность передавать поведение как параметр (callback) или создавать таблицы функций для выбора алгоритма во время выполнения.

 Синтаксис: 
```c
// объявление: тип_результата (*имя)(типы_параметров)
int (*op)(int, int);

// присваивание
op = add;

// вызов
int res = op(5, 3);
```

   Функции с переменным числом аргументов
Такие функции объявляются с многоточием `...` в конце списка параметров. Для доступа к аргументам используются макросы из `<stdarg.h>`: `va_list`, `va_start`, `va_arg`, `va_end`. Обязателен хотя бы один фиксированный параметр (например, `count` или форматная строка), по которому определяется количество и типы следующих аргументов.

 Пример: 
```c
 include <stdarg.h>

int sum(int count, ...) {
    va_list args;
    va_start(args, count);
    int total = 0;
    for (int i = 0; i < count; i++)
        total += va_arg(args, int);
    va_end(args);
    return total;
}
```

   Аргументы командной строки
При запуске программы параметры передаются в функцию `main` через `argc` (количество аргументов) и `argv` (массив строк). `argv[0]` — имя программы, `argv[1]` — первый аргумент, …, `argv[argc-1]` — последний, `argv[argc]` равно `NULL`. Аргументы всегда являются строками, поэтому числа нужно преобразовывать (например, с помощью `ft_atoi`).

  Задание

Вам необходимо разработать программу в соответствии с  одним из четырёх вариантов  (вариант выдаётся преподавателем или выбирается по списку). Программа должна:

- содержать хотя бы одну  вариативную функцию  (`...`);
- выводить результат через функции `ft_putchar_fd`, `ft_putstr_fd`, `ft_putnbr_fd`, `ft_putendl_fd` (или другие из libft);

   Общие требования ко всем вариантам
- Код должен компилироваться с флагами `-Wall -Wextra -Werror`.
- Не должно быть утечек памяти (проверка через `valgrind`).
- Все указатели, полученные через `malloc`/`ft_calloc`, должны освобождаться.
- При ошибках (недостаток аргументов, некорректные данные) программа должна выводить сообщение в `stderr` (файловый дескриптор 2) и завершаться с ненулевым кодом возврата.
- Для вывода сообщений об ошибках используйте `ft_putstr_fd("Сообщение", 2)`.

  Варианты заданий

   Вариант 1. Калькулятор с произвольным числом операндов

 Описание:   
Программа принимает название арифметической операции и список целых чисел. Выполняет операцию над всеми числами и выводит результат.

 Формат вызова: 
```
./calc <операция> [числа...]
```
- `<операция>`: `sum` (сумма), `mul` (произведение), `avg` (среднее арифметическое, целое, округлённое вниз).
- `[числа...]`: одно или несколько целых чисел.

 Реализация: 
1. Создать массив структур, содержащих имя операции и указатель на функцию-обработчик. Каждый обработчик имеет тип `int (*)(int, ...)`.
2. В `main`:
   - проверить `argc >= 3` (имя программы, операция, хотя бы одно число);
   - найти обработчик по имени;
   - преобразовать числа из `argv[2..argc-1]` в `int` с помощью `ft_atoi`;
   - вызвать обработчик, передав количество чисел и сами числа;
   - вывести результат через `ft_putnbr_fd`.
3. Если операция не распознана или чисел нет — вывести ошибку.

 Пример работы: 
```
$ ./calc sum 1 2 3
6
$ ./calc mul 2 3 4
24
$ ./calc avg 10 20 30
20
```

   Вариант 2. Фильтрация строк по критериям

 Описание:   
Программа применяет фильтр к набору строк. Строки могут быть переданы как аргументы или считаны из стандартного ввода.

 Формат вызова: 
```
./filter <критерий> [параметры] [строки...]
```
Если строки не указаны, они читаются из `stdin` построчно до EOF.

Критерии:
- `len <N>` — оставить строки, длина которых больше `N` (N — целое число).
- `start <подстрока>` — оставить строки, начинающиеся с заданной подстроки.
- `digit` — оставить строки, состоящие только из цифр.

 Реализация: 
1. Реализовать вариативную функцию `int match(const char *str, int criterion, ...)`, которая возвращает 1, если строка соответствует критерию, иначе 0. Внутри с помощью `va_list` извлекать дополнительные параметры (порог длины, подстроку и т.п.).
2. Создать массив структур, связывающих название критерия и соответствующий указатель на функцию-предикат. Предикат должен вызывать `match` с нужными аргументами или быть самостоятельным обработчиком.
3. В `main`:
   - разобрать `argc` и `argv`;
   - определить критерий и его параметры;
   - если строки переданы в аргументах — обработать их;
   - если нет — читать строки из `stdin` (используя `read` или `get_next_line`, но лучше использовать `ft_split` для разбиения всего ввода по `\n`);
   - для каждой строки применить выбранный предикат и при успехе вывести её через `ft_putendl_fd`.

 Пример работы: 
```
$ ./filter len 2 apple banana hi a
apple
banana
hi
$ ./filter start hel hello world helpful
hello
helpful
$ echo -e "123\nabc\n456\nxyz" | ./filter digit
123
456
```

   Вариант 3. Статистика числового набора

 Описание:   
Программа вычисляет статистические показатели для набора целых чисел, переданных в командной строке.

 Формат вызова: 
```
./stats <метрика> [числа...]
```
Метрики:
- `min` — минимальное значение;
- `max` — максимальное значение;
- `sum` — сумма;
- `avg` — среднее арифметическое (целое, округлённое вниз).

 Реализация: 
1. Реализовать вариативную функцию `int compute(int metric, int count, ...)`, где `metric` — код метрики (0..3). Внутри использовать `va_list` для перебора чисел и возвращать результат.
2. Вместо кодов можно использовать массив указателей на функции, каждая из которых имеет сигнатуру `int (*)(int, ...)`. По строковому имени выбирать нужный обработчик.
3. В `main`:
   - проверить `argc >= 3`;
   - преобразовать числа из `argv[2..argc-1]` в `int`;
   - выбрать функцию по имени метрики;
   - вызвать её, передав количество чисел и сами числа;
   - вывести результат через `ft_putnbr_fd`.
4. При пустом списке чисел или неизвестной метрике — вывести ошибку.

 Пример работы: 
```
$ ./stats min 5 3 8 1 9
1
$ ./stats max 5 3 8 1 9
9
$ ./stats sum 5 3 8 1 9
26
$ ./stats avg 5 3 8 1 9
5
```

   Вариант 4. Поиск строк по шаблону

 Описание:   
Программа ищет в наборе строк (аргументы или `stdin`) строки, удовлетворяющие заданному шаблону поиска.

 Формат вызова: 
```
./search <режим> <шаблон> [строки...]
```
Режимы:
- `substr` — строки, содержащие подстроку `шаблон`;
- `prefix` — строки, начинающиеся с `шаблон`;
- `suffix` — строки, заканчивающиеся на `шаблон`;
- `len` — строки, длина которых равна числу, заданному в `шаблоне` (шаблон преобразуется в число).

Если строки не переданы, они читаются из `stdin` построчно.

 Реализация: 
1. Создать массив структур, связывающих режим и указатель на функцию-предикат. Предикат принимает строку и шаблон (для `len` — целое число) и возвращает 1 при совпадении.
2. Предикаты должны использовать функции libft: `ft_strnstr`, `ft_strncmp`, `ft_strlen` и т.д.
3. В `main`:
   - проверить `argc >= 3` (имя, режим, шаблон);
   - по режиму выбрать предикат;
   - для `len` преобразовать шаблон в число через `ft_atoi`;
   - обработать строки из аргументов или из `stdin`;
   - для каждой строки применить предикат и при успехе вывести её через `ft_putendl_fd`.
4. Обработать ошибки: неизвестный режим, недостаточно аргументов.

 Пример работы: 
```
$ ./search substr abc abcde xyz abc123
abcde
abc123
$ ./search prefix pre prefix pred prelude
prefix
pred
prelude
$ ./search suffix ing running swim jump
running
$ ./search len 3 cat dog bird
cat
dog
```

```
lab4-variadic/solutions/student-[номер_студента]

│   └── libft.h            заголовочный файл libft (можно ссылаться на внешний)
├── src/
│   └── main.c             основная программа
├── Makefile
└── README.md              описание работы
```

Если libft не установлена системно, можно включить её в проект:

```
lab4-variadic/solutions/student-[номер_студента]
├── libft/                 исходники libft
├── inc/
│   └── libft.h
├── src/
│   └── main.c
├── Makefile
└── README.md
```

В `Makefile` должны быть цели:
- `all` — сборка программы;
- `clean` — удаление объектных файлов;
- `fclean` — удаление объектных файлов и исполняемого файла;
- `re` — пересборка.

---

  Пример Makefile

```makefile
NAME = calc        или filter, stats, search в зависимости от варианта
CC = gcc
CFLAGS = -Wall -Wextra -Werror -I./inc
LIBFT_DIR = ./libft
LIBFT = $(LIBFT_DIR)/libft.a

SRC_DIR = src
SRCS = $(SRC_DIR)/main.c
OBJS = $(SRCS:.c=.o)

all: $(NAME)

$(NAME): $(OBJS) $(LIBFT)
	$(CC) $(CFLAGS) $^ -o $@

$(LIBFT):
	make -C $(LIBFT_DIR)

%.o: %.c
	$(CC) $(CFLAGS) -c $< -o $@

clean:
	rm -f $(OBJS)
	make -C $(LIBFT_DIR) clean

fclean: clean
	rm -f $(NAME)
	make -C $(LIBFT_DIR) fclean

re: fclean all

.PHONY: all clean fclean re
```
